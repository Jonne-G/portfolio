 The material-browser is an interface between the material-database on disk, and the user’s scene. Its main adoption has been in maya, but it is made to be easily adapted to other applications supporting Qt (described in the code-overview below).

Before the interface, the users had no easy way of importing materials into their scene without using an external application. Initially the material browser was only intended to import materials, but throughout development it also became the defacto tool for managing materials within the scene, and on objects (the scene- and selection tab).

Making this interface was my first introduction to Qt and UI programming, and over the various iterations the browser went through, I learned a lot about the various quirks and features of Qt, and how to maintain large python applications.

Looking back and future plans

This tool is quite dear to me, it was among the first tasks I was assigned, and throughout the various iterations I made, it was essentially a benchmark for my coding skills. In its current version (v3) I'm quite happy with how it turned out.

The future development of this tool is an interesting one.
Initially, it was designed to be used as an asset-browser not just containing materials.
Additionally, the tool was set to be used in various DCCs, not just Maya.
Because of this, all underlying code was strictly written in "pure python" without any DCC-side code. All DCC logic was delegated to interaction modules as mentioned above. When I get back to this tool I'd love to realize this cross-DCC compatibility as I think it would be an interesting challenge.
